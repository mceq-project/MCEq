Fixed a reference cycle in `MCEqRun` that kept old instances alive across loop
iterations. `_restore_initial_condition` stored bound methods of `self`
(`self.set_primary_model`, etc.), closing a cycle `MCEqRun → list → tuple →
bound method → MCEqRun` that refcount alone could not break. On the Accelerate
backend this exhausted the fixed-size sparse matrix store (`SIZE_MSTORE=10`)
after ~5 iterations of constructing new `MCEqRun` objects, raising
`Exception: Matrix creation failed.` Now stores the method name as a string and
looks it up via `getattr(self, name)` at replay time, so old instances are freed
immediately by reference counting.
