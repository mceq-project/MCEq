Fixed `EarthGeometry.set_h_obs` to keep `r_obs` and `theta_max` consistent after
an altitude change: previously only `h_obs` was stored, leaving derived geometry
quantities stale. Added bounds validation in both `__init__` and `set_h_obs` so
that out-of-range observation heights (negative or above the top of atmosphere)
raise a `ValueError` immediately. Also corrected the long-standing location name
typo `SanGrasso` → `GranSasso` (it's Gran Sasso d'Italia, not wherever San Grasso
is); passing the old name now raises a helpful `ValueError`. Fixed a wrong file
existence check in `HDF5Backend` that tested `had_fname` instead of `em_fname`
when `config.enable_em` is set, causing a misleading error when the EM database
file was missing.
